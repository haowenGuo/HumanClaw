OpenClaw 边界
为了和 OpenClaw 解耦，HumanClaw 不再写 provider、模型、权限策略，也不会帮你安装、拉起、修复 OpenClaw runtime。
HumanClaw 负责什么
桌宠展示、聊天 UI、语音链路、后端模式切换，以及对用户本地 OpenClaw Gateway 的轻量桥接。
OpenClaw 自己负责什么
session、工具调用、任务执行、provider 与 model 配置、权限策略，以及所有高风险能力的真正启用。
