HumanClaw First Run
一次配置完成后，桌宠就能按你的节奏工作。
这不是老式安装页，而是首启向导。对话后端、镜头、语音和本地 OpenClaw 连接边界都在这里一次收口，后面都能从控制面板继续调。
Step 1
选择后端
先决定走陪伴服务，还是走你自己安装的本地 OpenClaw。
Step 2
桌宠与语音
把缩放、镜头、语音和开机形态一次调顺。
Step 3
连接目标
把两个地址定稳，再检查当前机器有没有已经跑起来的 OpenClaw。
你希望这台 HumanClaw 主要接哪条后端？
如果你更偏陪伴和聊天，就直接用陪伴后端；如果你已经自己装好了 OpenClaw，就切到本地 OpenClaw 模式。

开机自启
更适合把 HumanClaw 当作长期常驻的桌面伙伴。
显示在任务栏
如果你更偏桌面陪伴，可以关闭任务栏占位，只保留托盘和人物本体。
本地语音识别
开启后聊天窗可直接录音转文字，再交给当前选择的对话后端。
本地语音播报
把助手结果做成简短播报回到人物身上，更像随身桌宠。
最后定一下连接目标和健康检查
HumanClaw 不再管理 OpenClaw 的 provider、模型和权限。这里仅保存陪伴后端地址与本地 OpenClaw Gateway 地址，并检查当前能不能连通。
等待健康检查
保存当前选择后，可直接检查本机 OpenClaw Gateway 是否已经打通。
HumanClaw 负责
桌宠前端、聊天入口、语音识别与播报，以及对用户本地 OpenClaw Gateway 的桥接。
OpenClaw 负责
provider、模型、权限策略、session 与工具调用。要用高能力，就去 OpenClaw 自己那边配置。
完成后会直接写入本机配置，并且以后都能从控制面板继续调整。
